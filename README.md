# pySAred_EH

This repository contains extended versions of pySAred prepared by Dr. Elhoucine Hadden, based on the original pySAred software authored by Dr. Alexey Klechikov.

The repository preserves the first `v1.0.1` release, which documents the main contributions compared to the original pySAred code, and also provides the newer `v1.1.0` release with support for SuperADAM data generated by the Nomad interface starting from year 2026.

## Main updates by version

### pySAred_EH v1.0.1

A detailed update summary is available here:

`docs/pySAred_EH_v1.0.1_update_summary.txt`

Main highlights include:

- support for data generated at the MiniADAM instrument,
- sample-shape-aware overillumination correction,
- improved Single File Mode consistency with batch-mode behavior,
- sample-curvature correction reversibility fix,
- improved numerical safety checks,
- layout-managed GUI refactor,
- improved small-screen usability,
- Monitors / Time / slits preview and export,
- detector-image cursor/slider navigation.

### pySAred_EH v1.1.0

A detailed update summary is available here:

`docs/pySAred_EH_v1.1.0_update_summary.txt`

Main highlights include:

- support for SuperADAM Nomad NeXus `.nxs` files,
- preservation of the legacy `.h5` workflow,
- dynamic unpacking of Nomad scanned variables from their labels,
- configurable detector/scanned-variable unpacking for NR, NSF and PNR workflows,
- support for `Monitor1` normalization in Nomad files (initially absent in .nxs files during the upgrade, but later added), with documented fallback behavior,
- improved Nomad ROI and slit handling,
- automatic data-reduction summary file generation,
- improved Nomad unpacking dialog usability.

## Which version should I use?

Use `v1.1.0` or later for data acquired with the new SuperADAM Nomad interface (`.nxs` files).

Use `v1.0.1` if you need to reproduce or compare reductions performed with the earlier public pre-Nomad version.
