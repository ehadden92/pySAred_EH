# pySAred_EH v1.1.0 - Nomad .nxs support for SuperADAM

This release adds support for SuperADAM data generated by the Nomad interface while preserving the legacy pySAred_EH `.h5` workflow.

## Recommended use

Use `v1.1.0` for current SuperADAM Nomad `.nxs` data.

Use `v1.0.1` if you need exact continuity with the previous public pre-Nomad release.

## Main changes

- Added support for SuperADAM Nomad NeXus `.nxs` files.
- Preserved support for legacy `.h5` workflows.
- Added dynamic Nomad scanned-variable unpacking using labels stored in the file.
- Added configurable detector/scanned-variable unpacking for:
  - no unpacking / NR,
  - two-state NSF unpacking,
  - four-state PNR unpacking.
- Added `Monitor1` support for Nomad monitor normalization.
- Added fallback monitor behavior with a user warning when `Monitor1` is missing.
- Improved Nomad ROI and slit handling.
- Added a separate data-reduction summary file during export.
- Improved the Nomad unpacking dialog readability and small-screen usability.
- Updated documentation for version-specific usage.
