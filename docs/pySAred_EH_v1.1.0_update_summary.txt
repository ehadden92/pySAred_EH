pySAred_EH v1.1.0 update summary
=================================

This document summarizes the main updates introduced in pySAred_EH v1.1.0 compared to pySAred_EH v1.0.1.

pySAred_EH v1.1.0 keeps the legacy pySAred_EH workflow available for pre-Nomad data, while adding support for SuperADAM data generated by the Nomad interface.

Main additions
--------------

1. Support for SuperADAM Nomad NeXus files

pySAred_EH v1.1.0 adds support for data files generated by the new SuperADAM Nomad interface using the NeXus `.nxs` format.

The legacy `.h5` workflow remains available.

2. Nomad scanned-variable unpacking

The code can unpack scanned variables stored under:

/entry0/data_scan/scanned_variables/data

using their labels from:

/entry0/data_scan/scanned_variables/variables_names/label

This allows the loader to handle Nomad files in which the number or order of scanned variables changes, provided the labels remain unique and correctly associated with the stored data.

3. Configurable detector-data unpacking

The code now supports configurable handling of detector frames for different experimental modes:

- no unpacking / NR,
- two-state NSF unpacking,
- four-state PNR unpacking.

For interleaved data, the same unpacking choice is applied consistently to detector data and frame-indexed scanned variables.

4. Nomad monitor handling

The Nomad loader now supports `Monitor1` as the preferred monitor channel when available.

If `Monitor1` is not present, fallback monitor channels may be used, and the user is informed when a fallback is applied.

5. Nomad ROI and slit handling

The code reads Nomad ROI and slit values from the SuperADAM Nomad file structure and formats slit values more cleanly in the interface and summary output.

6. Data reduction summary file

The export now creates a separate data-reduction summary file. This avoids changing the exported data-file structure while preserving a trace of the reduction choices and parameters.

The summary can include information such as:

- applied corrections,
- used values,
- ROI information,
- skipped points,
- monitor choice,
- Nomad unpacking mode and order, when applicable.

7. Improved Nomad unpacking dialog

The Nomad unpacking dialog was adjusted for better readability while remaining usable on smaller screens.

Compatibility notes
-------------------

- Use pySAred_EH v1.1.0 for current SuperADAM Nomad `.nxs` data.
- Use pySAred_EH v1.0.1 if you need exact continuity with the previous public pre-Nomad release.
- The same attribution, rights and permissions stated in the repository remain applicable.
